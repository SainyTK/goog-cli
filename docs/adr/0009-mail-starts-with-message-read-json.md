# Mail Starts With Message Read JSON and Attachment Download

The first GoogleMail surface exposes `goog mail list`, `goog mail search`, `goog mail read`, and `goog mail attachment download` over Gmail Messages, not Threads. `read` returns the raw Gmail API Message JSON first because Message payloads can contain nested MIME parts, headers, text and HTML bodies, and attachments; a human-friendly renderer can be added later without hiding the native Gmail structure from scripts. Attachment download is included from the first slice because raw Message JSON can expose attachment IDs without giving users a way to retrieve the payload. Downloads fail when the destination file already exists, matching the CLI's safe default for local file writes. Mail commands request only Gmail readonly scope in the first slice. Mail list and search stay bounded with `--limit`; they do not expose `--all` in the first slice because each listed Message requires an additional metadata fetch.
